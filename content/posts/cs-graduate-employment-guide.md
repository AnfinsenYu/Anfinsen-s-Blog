## 前言

最近看到一篇很有参考价值的文章——[计算机读研规划求稳版（就业导向）](https://www.yuque.com/ljx1230/kpcla1/qsr5uvko52ylln46)，作者双非本科、中科大硕士，入学时技术栈几乎为零，最终拿了腾讯、字节、百度、快手、华为等多家大厂 offer。

这篇文章的路线很务实：**不赌算法岗的高风险高回报，走开发岗的确定性路线**。我结合自己的理解和经验，重新梳理了一遍，删掉了一些过时的内容，补充了 2026 年的实际情况。

> 💡 **一句话总结**：读研时间有限，与其什么都学一点，不如把后端开发这条路走通走透。

---

## 先看全局：三年时间线

```
拟录取 ──→ 暑假 ──→ 研一上 ──→ 研一下 ──→ 研一暑假 ──→ 研二 ──→ 秋招/春招
  放松      基础      框架+项目   中间件+八股    实习       冲刺      offer
```

| 阶段 | 时间 | 核心任务 | 产出 |
|------|------|----------|------|
| 拟录取 → 暑假 | 4~6 月 | 放松 + 了解方向 | 环境搭好 |
| 暑期 | 7~8 月 | Java 基础 + MySQL 入门 | 能写基本程序 |
| 研一上 | 9~1 月 | Spring 全家桶 + 项目实战 | 独立开发 Web 应用 |
| 研一下 | 2~6 月 | 中间件 + 八股 + 算法 | 简历项目 + LeetCode 100 题 |
| 实习阶段 | 研一暑假起 | 日常实习 → 暑期实习 | 实习经历 + 转正机会 |
| 秋招/春招 | 8~10 月 / 3~4 月 | 全力冲刺 | 大厂 offer |

---

## Phase 1：拟录取 ~ 暑假（4~6 月）

**先爽玩。** 别给自己太大压力，这是你读研前最后的悠闲时光。

如果实在闲不住，可以做这几件事：

- **逛 GitHub**，关注几个优质开源项目，培养阅读源码的习惯
- **了解后端技术栈全貌**，心里有个大概的图谱（不用深入，知道有什么就行）
- **搭好开发环境**：IDEA、Maven、Git、Docker，开学直接用

> 不建议提前学太多，开学后节奏会很快，这段时间保持手感就够了。

---

## Phase 2：暑期（7~8 月）—— 打地基

### Java 基础

**推荐方式：看书 > 看视频**

视频课程适合零基础入门，但效率偏低。直接看书 + 写代码练习，遇到不懂的再搜视频：

- 重点掌握：面向对象、集合框架、异常处理、IO 流、多线程基础
- Swing、AWT 这些 GUI 章节直接跳过，用不上
- 推荐书：《Java 核心技术》或《Head First Java》

### MySQL 入门

- 学会基本的 CRUD 操作，能写 `JOIN`、`GROUP BY`、子查询
- 暂时不需要深入原理（索引优化、事务隔离级别后面再学）
- SQL 这东西，写多了自然就会了，别光看不练

> 🎯 **阶段目标**：能用 Java 写出基本程序，能用 SQL 操作数据库。够了，不要贪多。

---

## Phase 3：研一上学期（9~1 月）—— 最关键的学期

这是**拉开差距的分水岭**。这个学期结束后，你应该能独立开发一个完整的 Web 应用。

### 学习顺序

按这个顺序来，不要跳步：

1. **Maven**（1~2 天）—— 依赖管理和项目构建，学会就行
2. **MyBatis**（3~5 天）—— ORM 操作数据库，重点学 XML 映射和注解方式
3. **Spring 核心**（2~3 周）—— IOC 和 AOP 是重中之重，事务管理也要掌握
4. **SpringBoot**（1~2 周）—— 自动配置、Starter 机制、整合各种组件
5. **项目实战**（贯穿始终）—— 找一个完整项目跟着做一遍

### 项目推荐

- **苍穹外卖**：经典的 SpringBoot 入门项目，涵盖后端开发大部分场景
- 或者自己从零搭一个系统（博客、商城、管理系统），关键是要**完整地做完**

### Redis 入门

- 学会基本数据结构（String、Hash、List、Set、ZSet）和常用命令
- 理解缓存的概念和使用场景
- 先不急深入集群和持久化，后面八股阶段再补

### 消息队列入门

- 了解基本概念：生产者、消费者、队列、Topic
- 学一个主流 MQ（RabbitMQ 或 RocketMQ）
- 理解核心场景：异步解耦、削峰填谷

> 💡 **这个学期结束后**：你应该能独立开发一个简单的 Web 应用了。如果做不到，说明哪个环节没学扎实，赶紧补。

---

## Phase 4：研一下学期（2~6 月）—— 三线并行

这个阶段要同时推进三件事，可以并行：

### 线一：准备简历项目

找一个**有亮点的项目**放到简历上：

- 可以是业务项目（AI Agent 应用、RAG 系统、智能客服等）
- 也可以是技术深度项目（分布式缓存、消息平台、RPC 框架等）
- 关键是要能讲清楚：**问题背景 → 解决方案 → 取得的收益**

面试官不关心你用了什么技术，关心你**解决了什么问题**。

### 线二：八股文

八股文是绕不开的，但不要死记硬背，要**理解原理**：

| 方向 | 重点 |
|------|------|
| Java 基础 | JVM 内存模型、GC 算法、并发编程（锁、线程池、CAS） |
| 数据库 | 索引原理（B+ 树）、事务隔离级别、慢查询优化 |
| Redis | 数据结构实现、持久化机制、缓存穿透/击穿/雪崩 |
| 计算机网络 | TCP 三次握手/四次挥手、HTTP/HTTPS、DNS |
| 框架 | Spring IOC/AOP 原理、SpringBoot 自动配置流程 |
| 操作系统 | 进程 vs 线程、内存管理、死锁 |

### 线三：算法

- **LeetCode Hot 100** 刷 2~3 遍，应付面试够用了
- 重点掌握：双指针、滑动窗口、BFS/DFS、动态规划、二叉树
- 不需要刷到竞赛水平，能稳定解决中等难度题目即可
- 推荐用 Anki 或类似工具做间隔复习，防止刷了就忘

> ⚠️ **重要提醒**：项目 + 算法准备好了就可以投简历了。八股可以边面边背，永远准备不完的。

---

## 实习：最重要的加分项

有实习经历和没有，面试完全是两个世界。

### 日常实习（研一暑假或更早）

**前两个月：**
- 熟悉业务和代码 base，把分配的任务做好做扎实
- 不要急着表现，先把基础工作做漂亮

**中期：**
- 多探索，看看能否对项目做出改进
- 改进不一定是大优化，可以是组内提效工具、自动化脚本、文档完善

**末期：**
- 把自己做的有亮点的东西整理清楚
- 学习组内优秀方案，理解思路后可以写到简历上

### 暑期实习（3~4 月投递）

暑期实习不是暑假才开始，**3、4 月就要投了**。

有实习之后的面试，八股问得会少很多，大多围绕你的实习内容。描述实习经历的两个框架：

1. **问题 → 方案 → 收益**：先讲背景，再讲怎么解决，最后讲结果
2. **问题 → 方案对比 → 选择理由**：先讲背景，再对比各方案利弊，最后讲为什么选这个

> 🎯 这套说辞下来，面试官会觉得你很有工程思维。

## 转正 or 秋招

### 转正成功 🎉

恭喜！研三好好准备毕业论文，不要延毕。期间保持学习习惯，关注新技术。

### 转正失败 → 全力秋招

不要慌。你已经有了实习经验，这本身就是最大的资本。秋招是主战场，机会比春招多得多。

---

## 导师不放养怎么办

如果你的导师不放养，只能实习一段时间：

- **至少在研二下学期前**学完研一的所有内容
- 少在导师面前表现——能力越大活越多，活是干不完的
- 多偷偷学习技术，有条件的可以在本地找个小公司实习
- 拿到大厂 offer 后和导师商量去实习，态度可以强硬一点

这不是教你偷懒，是教你**合理分配时间**。读研的核心目标是就业，不是给导师打工。

## 2026 年的额外建议

原文写得扎实，但有些内容需要更新。补充几点：

### 1. AI 已经不是选修课

2026 年了，AI 不是加分项，是**基本素养**：

- 了解 LLM 的基本概念和使用方式
- 熟练使用 AI 辅助编程（Cursor、Claude Code 等），提升开发效率
- 如果有兴趣，可以深入 Agent 开发、RAG 等方向，这是新的增长点

### 2. 工程化能力不能缺

除了代码本身，这些也是基本功：

- **Git**：版本控制，分支管理，Code Review 流程
- **Docker**：容器化部署已经是标配
- **CI/CD**：了解 GitHub Actions 或 Jenkins
- **Linux 基本操作**：服务器上调试是日常

### 3. 建一个技术博客

通过写博客沉淀知识，面试时也是加分项。零成本方案：Hugo + GitHub Pages，你现在看的就是。

### 4. 身体是革命的本钱

读研压力不小，记得：
- 规律作息，不要长期熬夜
- 适当运动
- 心态放平，offer 拿多拿少都有出路

## 最后

这条路线的核心逻辑很简单：**集中精力，把后端开发这条路走通走透**。

不需要什么都学，不需要追最新技术栈。把 Java + Spring + MySQL + Redis + MQ 这条主线吃透，配合一个有亮点的项目和扎实的算法基础，大厂 offer 并不遥远。

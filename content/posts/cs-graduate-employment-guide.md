## 前言

最近看到一篇很棒的文章——[计算机读研规划求稳版（就业导向）](https://www.yuque.com/ljx1230/kpcla1/qsr5uvko52ylln46)，作者双非本科、中科大软件学院硕士，技术栈基础几乎为零，最终拿到了腾讯（大SP）、字节（小SSP）、百度（大SP）、快手（大SSP）、华为（大SSP）等多家大厂 offer。

这篇文章的路线非常务实，不追求算法岗的高风险高回报，而是走**开发岗的求稳路线**。我结合自己的理解和经验，整理出一篇更适合大多数同学参考的版本。

> 💡 **核心思想**：读研期间的时间是有限的，与其什么都学一点，不如集中精力把后端开发这条路走通走透。

---

## 时间线总览

| 阶段 | 时间 | 核心任务 |
|------|------|----------|
| 拟录取 → 暑假 | 4~6月 | 放松调整，了解方向 |
| 暑期 | 7~8月 | Java 基础 + MySQL 入门 |
| 研一上学期 | 9~1月 | Spring 全家桶 + 项目实战 |
| 研一下学期 | 2~6月 | 中间件 + 微服务 + 八股 + 算法 |
| 实习阶段 | 研一暑假 ~ 研二上 | 日常实习，积累经验 |
| 暑期实习 | 3~4月投递 | 争取转正机会 |
| 秋招/春招 | 8~10月 / 3~4月 | 全力冲刺 |

---

## 第一阶段：拟录取 ~ 暑假（4~6月）

**先爽玩！** 别给自己太大压力，这段时间很长，不急在一时。

可以做的事情：
- 了解一下后端开发的技术栈全貌，心里有个大概的图谱
- 逛逛 GitHub，关注一些优质开源项目
- 把开发环境搭好（IDEA、Maven、Git、Docker 等）

---

## 第二阶段：暑期（7~8月）

### Java 基础语法

**推荐方式：看书 > 看视频**

视频课程适合零基础入门，但效率偏低。推荐直接看书，配合代码练习：

- 看书中的 case 就基本会了，Swing 章节可以跳过
- 重点掌握：面向对象、集合框架、异常处理、IO 流、多线程基础

### MySQL 入门

- 学会基本的 CRUD 操作即可
- 暂时不需要深入原理（索引优化、事务隔离级别等后面再学）
- 推荐直接上手练习，SQL 这东西写多了自然就会了

> 🔑 **这个阶段的目标**：能用 Java 写出基本的程序，能用 SQL 操作数据库。够了，不要贪多。

---

## 第三阶段：研一上学期（9~1月）

这是**最关键的一个学期**，要把 Spring 全家桶学明白，并且通过项目实战把知识串起来。

### 学习路线

1. **Maven**：学会依赖管理和项目构建（1~2天够了）
2. **MyBatis**：学会用 ORM 操作数据库
3. **Spring 核心**：IOC 和 AOP 是重中之重，事务管理也要掌握
4. **SpringBoot**：自动配置、Starter 机制、整合各种组件
5. **项目实战**：找一个完整的项目跟着做一遍

### 推荐项目

- **苍穹外卖**：经典的 SpringBoot 入门项目，涵盖了后端开发的大部分场景
- 或者自己找一个业务系统（博客、商城、管理系统等）从零搭建

### Redis

- 学会基本数据结构和常用命令
- 理解缓存的概念和使用场景
- 先不急着深入集群和持久化

### 消息队列（MQ）

- 了解消息队列的基本概念（生产者、消费者、队列）
- 学一个主流的 MQ（RabbitMQ 或 RocketMQ）
- 理解使用场景：异步解耦、削峰填谷

> 💡 **这个学期结束后**：你应该能独立开发一个简单的 Web 应用了。

---

## 第四阶段：研一下学期（2~6月）

这个阶段要做的事情比较多，但可以**并行推进**：

### 准备项目

找一个有亮点的项目放到简历上：
- 可以是业务相关的项目（如 AI Agent 应用、RAG 系统等）
- 也可以是中间件深度使用项目（如分布式缓存、消息平台等）
- 关键是要能讲清楚**问题背景 → 解决方案 → 取得的收益**

### 八股文准备

八股文是绕不开的，但不要死记硬背，要理解原理：

| 方向 | 重点内容 |
|------|----------|
| Java 基础 | JVM 内存模型、GC、并发编程 |
| 数据库 | 索引原理、事务隔离、慢查询优化 |
| Redis | 数据结构、持久化、缓存穿透/击穿/雪崩 |
| 计算机网络 | TCP 三次握手/四次挥手、HTTP/HTTPS |
| 框架 | Spring IOC/AOP 原理、SpringBoot 自动配置 |

### 算法

- **LeetCode Hot 100** 刷 2~3 遍，应付面试够用了
- 重点掌握：双指针、滑动窗口、BFS/DFS、动态规划、二叉树
- 不需要刷到竞赛水平，能解决中等难度的题目即可

> ⚠️ **重要提醒**：项目和算法准备好了就可以投简历了，八股可以边背边投，永远准备不完的。

---

## 实习阶段

### 日常实习（研一暑假或更早开始）

实习是**最重要的加分项**，有实习经历和没有完全是两个世界。

**实习前两个月：**
- 熟悉业务和代码base
- 把分配的任务做好、做扎实

**实习中期：**
- 多探索新东西，看看能否对项目做出改进
- 改进不一定是大的优化，可以是组内提效工具、自动化脚本等

**实习末期：**
- 把自己做的有亮点的东西整理清楚
- 学习组内其他正式工做的优秀方案，理解思路后写到简历上

### 暑期实习（3~4月投递）

暑期实习并不是暑假才开始，往往 3、4 月就开始招聘了。

**有实习之后的面试**，八股文问得会相对少一些，大多围绕你的实习内容。描述实习经历的两个角度：

1. **问题 → 方案 → 收益**：先讲问题背景，再讲解决方案，最后讲取得的收益
2. **问题 → 方案对比 → 选择理由**：先讲问题背景，再对比各方案利弊，最后讲为什么选这个方案

> 🎯 这套说辞下来，面试官会觉得你很有工程思维和探索精神。

---

## 转正 or 秋招

### 转正成功 🎉

恭喜！研三可以好好准备毕业论文，不要延毕。期间可以持续关注新技术，保持学习习惯。

### 转正失败 → 全力秋招/春招

不要慌，按照暑期实习的经验继续准备。秋招是主战场，机会更多。

---

## 给导师不放养的同学

如果你的导师不放养，只能实习一段时间，建议：

- **至少在研二下学期前**学完研一的所有内容
- 少在导师面前表现——能力越大活越多，活是干不完的
- 多偷偷学习技术，有条件的可以在本地找个小公司实习
- 拿到大厂 offer 后和导师商量去实习，态度可以强硬一点

## 我的补充建议

原文的路线非常扎实，我再补充几点：

### 1. 重视工程化能力

除了代码本身，还要掌握：
- **Git**：版本控制是基本功
- **Docker**：容器化部署已经是标配
- **CI/CD**：了解 GitHub Actions 或 Jenkins
- **Linux 基本操作**：服务器上调试是日常

### 2. 建立自己的技术博客

通过写博客沉淀知识，面试时也是加分项。比如你现在看的这个博客就是用 Hugo + GitHub Pages 搭建的，零成本、高效率。

### 3. 关注 AI 趋势

2026 年了，AI 已经不是选修课：
- 了解 LLM 的基本概念和使用方式
- 尝试用 AI 辅助编程（Cursor、Claude Code 等）
- 如果有兴趣，可以深入 Agent 开发、RAG 等方向

### 4. 保持身体健康

读研期间压力不小，记得：
- 规律作息，不要长期熬夜
- 适当运动，身体是革命的本钱
- 心态放平，offer 拿多拿少都有出路

---

## 总结

这条路线的核心逻辑是：**集中精力，把后端开发这条路走通走透**。

不需要什么都学，不需要追最新的技术栈，把 Java + Spring + MySQL + Redis + MQ 这条主线吃透，配合一个有亮点的项目和扎实的算法基础，大厂 offer 并不遥远。
